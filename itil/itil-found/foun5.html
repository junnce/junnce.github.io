# 第五章：IT 服务的生命周期

# 一、简介

ITTL最佳实践，是世界范围内认可度最高的IT服务管理​指南，在 ITIL v3 中，服务的生命周期被划分为五个部分

开局一张图，内容全靠编

从图中内容我们可以清晰看到五个周期相互依赖的关系，侧重，以及周而复始形成持续化改进

# 二、周期详解

# 服务战略

服务战略（SS）,该部分核心为创造价值，开拓新领域创造营收的基础上，以此解决客户反馈痛点，考虑做与不做（即：指导如何设计、开发和实施服务管理，包括组织能力及战略资产.）

# 服务设计

服务设计（SD），该部分核心为设计价值，将需求量化设计，指导设计服务及服务管理流程

# 服务转换

服务转换（ST），该部分核心为指导将新的或变更的服务引入生产环境。由此可以看出这是个运维 & 开发协作的过程形成持续交付

# 服务运营

服务运营（SO）是整个 ITIL 服务管理中最为丰满的一环（PS：ITIL 最初是运维起家的，相关的流程经验很丰富），该部分主要核心为保证服务支持和交付的效率和效力，客户与服务提供者价值并重。

# 持续服务改进

持续服务改进（CSI），将齿轮部分想象为动能提供中心（周而复始），该部分主要核心为通过更好的设计、引入和运营为客户创造价值
